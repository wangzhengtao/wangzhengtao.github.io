项目经验

A+学车驾校管理平台(2018/10-2019/05)

  1. 项目描述
    该项目是整个SaaS系统的基础项目，主要提供学员报名服务、学员约课服务、驾校资源管理服务、驾校组织架构管理服务、驾校财务服务等。
  2. 项目技术
    Spring Cloud + Spring Boot + MyBatis + MySQL + Redis + MyCat
  3. 项目职责
  • 组织架构包括部门、职务、员工的设计及开发,使用36进制层级code明确标明了组织架构以及各部门产生业务数据的层级关系
  • 驾校资源包括场地、车辆、时段的设计及开发
  • 学习模块包括课程的自动生成、学员约课、签到等的设计及开发，使用Redis做缓冲区以减小自动生成课表时对内存和数据库产生的压力，并且使用Redis做分布式锁，解决了学员约课时的资源竞争问题
  • 项目的用户登陆体系的设计及开发，由于某些原因，没有使用认证授权的框架，加上是微服务的架构设计，因此抽离出来公共用户体系，用于做登陆认证，之后通过token令牌访问系统

计时平台(2016/10-2017/3)

  1. 项目描述
    该项目是应交通运输部2016年17号公告机动车驾驶员计时培训系统计时终端技术规范，开发的一个计时终端消息接收平台。该系统可以有效防止学员、教练作弊，学员、教练登陆采用扫码、刷卡、身份证、人脸识别四种登陆方式，训练途中有随机拍照验证。并且系统会最终验证行车途中的分钟学时、GPS轨迹信息、照片信息等判断学时是否有效。
  2. 项目技术
    Spring Boot + Spring-Data-JPA + Spring-Data-Mongo + Netty + Akka + Mongo + MySQL
  3. 项目职责
  • 学员登陆的设计及开发，包括了扫码登陆，人脸认证登陆，身份证刷卡登陆，学员卡刷卡登陆
  • 鉴权验证，获取学员、终端信息接口设计及开发，这个涉及到一些证书的加解密以及验证
  • 图片接收及验证接口设计及开发，这块的问题在于图片的分包与组合
  • 轨迹信息接收和判断接口设计及开发，起初我们将10秒一次的轨迹点存于数据库中，对数据库的压力过大，后来存到了百度鹰眼，解放了数据库
  • 分钟学时接收转发接口设计及开发，起初这个1分钟一次的分钟学时也是存放于数据库中的，同样对数据库造成很大压力，后改用Mongo存储

驾校计时监管平台(2017/5-2017/9)

  1. 项目描述
    该项目是应交通运输部与计时平台对应的计时监管平台项目，主要以帮助驾校规范学员的学习流程、规范教练员的教学方式为目的。该系统功能包括接收计时平台收集的学员学习数据，驾校注册备案，驾校场地、教练、车辆等资源的备案、计时终端备案、车辆绑定，计时终端消息下发等功能。
  2. 项目技术
    Spring Boot + Spring-Data-JPA + Spring-Data-Mongo + Netty + Akka + Mongo + MySQL
  3. 项目职责
  • 驾校注册备案、驾校学员、教练、计时终端备案接口的设计及开发，这些需要给第三方推送数据，使用了Spring的RestTemplatet发送Http请求
  • 计时终端内部流转的设计及开发，这个定义了一套流转的规则，比较繁琐，流转过程中还涉及到备案与删除备案等
  • 车辆、计时终端的绑定解绑设计及开发
  • 电子教学日志模块的设计及开发，学员登陆开始上课后就要生成电子教学日志，且后台要能实时查看与控制
  • 阶段学时生成及备案的设计及开发，这是个定时任务，每天按规则将满足条件的电子教学日志生成阶段学时记录并生成之后要流转（与第三方系统流转审核盖章）的PDF文件

期间也做过分销系统、积分商城、论坛、CMS、优惠券、保险理赔等其他的一些项目。
